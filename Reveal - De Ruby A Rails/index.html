2. Le langage

  • Se veut orienté objet (Python l'était mais pas assez pour « Matz »)

3. Exemples Code

Une classe objet

							
class Vehicule
  def initialize(couleur, marque)
    @couleur = couleur
    @marque = marque
  end
  protected :afficheDescription
  #Seule afficheDescription est en protected
  def afficheDescription
    puts "Véhicule #{@couleur} de la marque : #{@marque}"
  end
end

Un héritage

							
class Voiture < Vehicule
  def initialize(couleur, marque, type)
    super(couleur, marque)
    @type = type
  end
  private
  #toutes les methodes qui suivent sont privées
  def afficheType
  	puts "Le type de cette voiture est #{@type}"
  end
end

2. Projet

Un projet sous Rails se compose de la manière suivante :

  • app/ dossier dans lequel nous placerons nos controllers / views / models / helpers / ...
  • bin/ contient les scripts de base de rails (lancement, déploiement, ... )
  • config/ règles de lancement par environnement / routing / connexion à la BDD / internationnalisation ...
  • config.ru fichier qui start l'app et renvoie sur les fichiers de config/
  • db/ contient le schema de la BDD
  • Gemfile & Gemfile.lock les deux fichiers gèrent les dépendances Gem que l'on a besoin pour notre app Rails

2. Projet (2)

  • lib/ dossier qui contiendra les différents modules installés
  • public/ fichiers compilés / statics
  • Rakefile fichier qui recense les tasks (liées à lib/tasks) que l'on peut éxécuter en ligne de commande (ex : tâches de migration)
  • test/ regroupe les tests unitaires, fixtures, ...
  • tmp/ cache, ...
  • vendor/ applications tierces (y comprit les installations des applications venues de gem)
